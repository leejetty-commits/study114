# 릴리즈 메모 — A28-07 노출·권한 보정 (운영 투입)

**일자:** 2026-07-07  
**대상:** 내부 운영자 (`role_type=admin`)  
**환경:** staging / production — home-ui admin 라우트 + `/api/admin/*`

---

## 한 줄 공지 (운영자용)

> **제출됨(`submitted`) 상태 제출은 `#/admin/exposure`에서 공개할 수 없습니다.**  
> `#/admin/submission-docs`(A28-06)에서 **노출 반영**하세요.

---

## A28-07 변경점

| 항목 | 내용 |
|------|------|
| 화면 | `#/admin/exposure` — 노출·권한 수동 보정 |
| API | `GET/PATCH /api/admin/exposure.php` |
| 대상 | 공부방 · 과외쌤 · 제출 (`study_room` / `tutor` / `submission`) |
| 조치 | `hide` · `publish` · `inquiry_status`(공부방만) |
| 로그 | `admin_operation_logs` — `action_kind` · `target_id` · `detail_memo` |
| 인증 | 세션 `role_type=admin` · `operator_id`는 세션 email (클라이언트 위조 무시) |

**사용자-facing 금지 유지:** 승인 · 반려 · 검증 완료 · 인증 — 운영 UI·API 응답에 포함하지 않음.

---

## A28-06 / A28-07 경계

| 상황 | 사용 화면 | API | 조치 |
|------|-----------|-----|------|
| 제출 **제출됨** → 게시중 | **A28-06** `#/admin/submission-docs` | `PATCH /api/admin/submission-queue.php` | `expose` |
| 제출 **게시중** ↔ **숨김** 보정 | **A28-07** `#/admin/exposure` | `PATCH /api/admin/exposure.php` | `publish` / `hide` |
| 공부방·과외 **공개중** ↔ **숨김** | A28-07 | 동일 | `publish` / `hide` |
| 공부방 **상담 상태** 보정 | A28-07 | 동일 | `inquiry_status` |

**강제 규칙 (2026-07-07):** `submitted` 제출에 A28-07 `publish` 요청 시 **HTTP 422**

```
제출됨 상태는 A28-06 제출자료 큐에서 노출 반영하세요. (submitted → published는 A28-07 publish 불가)
```

## 운영 로그 조치 구분 (`#/admin/logs`)

| `action_kind` | 화면 라벨 | 대상 유형 | 의미 |
|---------------|-----------|-----------|------|
| `hide_profile` | 프로필 숨김 | 공부방 · 과외쌤 | 검색·노출 숨김 (`user_notified` 기본 true) |
| `exposure_correction` | 노출 보정(공개·상담) | 공부방 · 과외쌤 | 공개 복원 또는 `inquiry_status` 변경 |
| `submission_expose` | 제출 노출 반영 | 제출 | 게시중 — A28-06 큐 또는 A28-07 보정 |
| `submission_hide` | 제출 숨김 | 제출 | 비공개 처리 |

로그 목록에 **대상 유형** 열 추가 — `target_id`만으로 혼동 줄임.

## 스모크 체크리스트 (staging/운영)

- [ ] `ops@…` (또는 운영 admin) 로그인
- [ ] `#/admin/exposure` 접근 · 목록·필터 로드
- [ ] 필터: 유형=제출 · 상태=제출됨 → **공개중 버튼 없음** · `→ A28-06 노출 반영` 링크 표시
- [ ] 동일 row에서 API `publish` 시도 시 **422** · 위 메시지 확인
- [ ] `#/admin/submission-docs`에서 동일 제출 **노출 반영** 성공
- [ ] `#/admin/logs`에서 최근 조치 4종 라벨·대상 유형 구분 가능

**자동화:** `npm run e2e:ops-smoke` · 전체 회귀 `npm run e2e:a28-07`

## 헷갈리기 쉬운 케이스

1. **제출이 제출됨인데 exposure에서 공개하고 싶다** → A28-06으로 이동 (exposure는 링크 제공)
2. **이미 게시중인 제출을 숨기고 싶다** → A28-07 또는 A28-06 `hide` (published만)
3. **로그에 submission_expose가 두 API에서 찍힌다** → 정상. `target_id`=`submission:{post_key}` · 대상 유형=제출로 확인
4. **exposure_correction이 공개인지 상담인지** → `detail_memo` · 당시 화면 조치 참고 (공개=publish · 상담=inquiry_status)
