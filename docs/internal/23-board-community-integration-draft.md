# 23장 — [내부 초안] 게시판·자료실·그누보드 커뮤니티 통합 그림

**상태:** 내부 기획 초안 (SSOT 미잠금) · **2026-07-04 정의 보강**  
**작성:** 2026-07-04  
**전제 SSOT:** 2·4·6·17장 · **18장=유료·결제(별도)** · 본 문서=**23장**  
**코드:** 본 문서는 기획만 — 구현 범위·일정은 10장에서 별도 편입

> **장 번호:** 게시판·자료실·GNU 커뮤니티 연동은 **23장**. **18장**은 기존 SSOT대로 **유료·결제·entitlement** 전담(15·16·17장 연동).

## 0. 한 줄 요약

**Study114(본서비스+게시판/자료실)** 와 **그누보드 커뮤니티** 는 **아예 별도 사이트·별도 운영**이다.  
**게시글·자료·파일·DB 콘텐츠는 전혀 공유하지 않는다.**  
연동은 **로그인·신원(기초 회원정보)만** — 회원 편의와 기득권(한 번 가입으로 양쪽 이용) 목적.  
고객센터와 자료실은 **Study114 안**에서만 Board Engine `boardKey`로 구분한다.

### 0-1. 잠금 원칙 (운영자 합의)

| 원칙 | 내용 |
|------|------|
| **콘텐츠 비공유** | 공지·FAQ·자료실 ↔ 커뮤니티 글/첨부 **연동·미러·위젯·크로스링크 DB 없음** |
| **별도 사이트** | URL·DB·관리자·배포 **독립** |
| **연동 범위** | **로그인·신원만** (4장 `users` 정본 → GNU `g5_member` 파생) |
| **커뮤니티 권한** | 후순위: Study114 역할과 무관하게 **무료회원 / 유료회원 2단** 단순 매핑 → 커뮤니티에서 **동일 권리** 부여 가능 (마케팅·기득권) |

## 1. 큰 그림 (2사이트 + 신원만 연결)

```
┌──────────────────────────────┐         ┌──────────────────────────────┐
│   Study114 사이트 (독립)      │         │   커뮤니티 사이트 (독립)        │
│   study114.net               │         │   community.* (그누보드)      │
├──────────────────────────────┤         ├──────────────────────────────┤
│ 본서비스 · auth · DB(4장)    │         │ g5_* 전용 DB                 │
│ Board Engine                 │   ╳     │ 게시판 · 댓글 · 첨부 · 신고    │
│  └ notice/faq/guide/library │ 콘텐츠   │ g5_member                    │
│ 고객센터(17) · (자료실)       │  공유    │                              │
│ 쪽지(16) · 등록·비교          │  없음    │                              │
└──────────────┬───────────────┘         └──────────────┬───────────────┘
               │                                        │
               └──────────── SSO / 로그인·신원만 ─────────┘
                         users (정본) → g5_member (파생)
                         + 후순위: 무료|유료 → mb_level 단순 2단
```

| 사이트 | 역할 | 콘텐츠 |
|--------|------|--------|
| **Study114** | 매칭·등록·비교·쪽지·고객센터·자료실 | **자체 Board Engine만** |
| **그누보드** | 회원 커뮤니티 | **GNU 게시판만** |

> **판단:** 양쪽은 **완전 별도 운영**. 연동은 **「같은 사람으로 로그인」** + (후순위) **커뮤니티 등급 2단** 뿐.

## 2. 회원 연동 — 로그인·신원만 (콘텐츠·역할 상세 비공유)

### 2-1. 정본·파생

| 시스템 | 회원 테이블 | 정본 여부 |
|--------|-------------|-----------|
| Study114 | `users` + `user_profiles` | **정본 (Master)** |
| 그누보드 | `g5_member` | **파생 (Linked)** — 커뮤니티 전용 |

### 2-2. 동기화 필드 (로그인·신원만)

| 필드 | 방향 | 비고 |
|------|------|------|
| email (ID) | Study114 → GNU | 로그인 키 |
| name | S114 우선 | 표시명 |
| password | **동기화 안 함** | SSO만 |
| status | S114 → GNU | 활성/차단/탈퇴 |
| phone, gender, birth | 선택 | 기초 신원 |

**동기화하지 않을 것 (명시):**

- 게시글·댓글·첨부·자료실 파일
- 공부방/과외/학생 프로필·등록 데이터
- 쪽지·찜·비교·노출(Prime/Pick) 상태
- Study114 **역할별 세부 권한** (커뮤니티에 그대로 넘기지 않음)

### 2-3. 커뮤니티 권한 — 단순 2단 (후순위·마케팅)

Study114 내부는 역할·유료등록이 복잡해도, **커뮤니티에서는 단순화**:

| S114 매핑 (로직 예시) | GNU `mb_level` | 커뮤니티 권리 |
|----------------------|----------------|---------------|
| 비로그인 | — | 읽기만 또는 가입 유도 |
| 로그인 (무료) | `1` 무료회원 | 글쓰기·댓글·첨부 (동일) |
| 유료 (공급자 유료등록 등 조건) | `2` 유료회원 | **무료와 동일 권리** + 뱃지·노출만 차별 가능 |

> **의도:** 커뮤니티에서 **「유료도 무료도 같은 권한」** — 기득권·마케팅 («본서비스 유료 = 커뮤니티도 우대 체감»)  
> 실제 차별은 포인트·뱃지·프로필 장식 등 **가벼운 요소**로, 쓰기 권한 자체는 맞춤.

### 2-4. 연동 테이블 (Study114 측)

```
member_external_links
  - id
  - user_id          → users.id
  - provider         → 'gnu_board5'
  - external_id      → g5_member.mb_id
  - last_synced_at
  - sync_status
```

### 2-5. 로그인 흐름 (목표)

```
[사용자] Study114 로그인 (auth)
    → 세션/JWT 발급
    → 커뮤니티 진입 시 signed token 또는 SSO endpoint
    → 그누보드 custom auth / 연동 플러그인
    → g5_member 자동 생성·갱신 (기초정보만)
```

| 방식 | 장점 | 단점 |
|------|------|------|
| **A. SSO 토큰 (권장)** | 비밀번호 이중관리 없음 | 그누보드 커스텀 개발 |
| B. 비밀번호 미러링 | 구현 단순 | 보안·탈퇴 동기화 지옥 |
| C. 완전 분리 계정 | 독립 | UX 나쁨 · 연동 목적 상실 |

**1차:** Study114 로그인 → 커뮤니티 진입 시 **자동 로그인(SSO)**  
**UI:** 본서비스 GNB에 「커뮤니티」= **외부 사이트 링크** (임베드·iframe·글 API 없음)

## 3. Board Engine — 하나의 엔진, 여러 채널

### 3-1. boardKey 목록 (확장)

| boardKey | 용도 | 노출 위치 | 성격 |
|----------|------|-----------|------|
| `notice` | 공지 | 17장 고객센터 | 운영 · 단일 펼침 |
| `faq` | FAQ | 17장 고객센터 | 운영 · 다중 펼침 |
| `safe-guide` | 안전과외 | 17장 고객센터 | 운영 · 아코디언 · 1차/후순위 |
| `policy-log` | 약관 변경 이력 | 17장·푸터 | 운영 · 후순위 |
| **`library`** | **자료실** | **유틸/GNB (후순위)** | **운영 큐레이션 자료** |
| `library-template` | 양식·체크리스트 | 자료실 하위 | 다운로드 중심 |
| `library-guide-pdf` | 가이드 PDF | 자료실·고객센터 링크 | 파일+메타 |

### 3-2. 고객센터 vs 자료실 — 같은 엔진, 다른 규칙

| 구분 | 고객센터 보드 | 자료실 (`library`) |
|------|---------------|-------------------|
| 목적 | 신뢰·안내·정책 | 학습·운영 **다운로드·참고** |
| 작성 주체 | 운영자만 | 운영자 (+ 후순위 검증된 공급자?) |
| UI 패턴 | 아코디언·게시판형 | **카드/리스트 + 썸네일 + 다운로드** |
| 첨부 | 후순위 | **핵심 (PDF, HWP, 이미지)** |
| 역할 노출 | 전체/역할별 | 학부모·공부방·과외 **필터** |
| 댓글 | 없음 | 없음 (1차) |

> 자료실·고객센터 글은 **Study114 DB에만** 존재. 커뮤니티와 **ID·URL·파일 공유 없음**.

### 3-3. 자료실 메뉴 (6장과의 관계)

- 6장: GNB **「자료실/포털 대확장」 1차 제외**
- **급한 데로:** 유틸 메뉴 또는 고객센터 인근 **「자료실」** 1개 → `library` 보드
- **대확장(후순위):** 카테고리·검색·인기자료·역할별 추천

```
[유틸] 자료실  →  #/library  (home-ui 또는 전용 라우트)
                    → Board Engine library 보드
                    → 파일 다운로드 (CDN/스토리지)
```

## 5. 관리자 — 운영 허브 분리

```
[Admin Study114]
  ├── 회원·역할·등록 심사
  ├── Board Engine 관리  ← 고객센터 + 자료실 (통합 CMS)
  │     ├── 보드별 글 CRUD
  │     ├── 예약발행·핀·역할 노출
  │     └── library 파일·카테고리
  └── (링크) GNU 관리자  ← 커뮤니티 게시판·회원·신고

[Admin GNU]
  └── 커뮤니티 전용 (그누보드 admin)
```

**운영 묘:** Study114 admin에서 **「오늘 발행할 공지·자료」** 한 화면 + GNU admin은 **커뮤니티 모더레이션**.

## 6. 데이터·API 경계 (콘텐츠 격리)

```
[Study114] — 자체 닫힌 영역
  /api/board/...          ← notice, faq, library (GNU API 호출 없음)
  board_posts, files      ← Study114 DB only

[GNU] — 자체 닫힌 영역
  /bbs/...                ← 커뮤니티 only
  g5_write, g5_board      ← GNU DB only

[유일한 공통 통로]
  /api/sso/issue-token    ← S114 로그인 후 커뮤니티용 짧은 토큰
  /api/sso/verify         ← GNU 플러그인이 검증 → g5_member 세션
  member tier sync        ← 무료|유료 → mb_level (배치/이벤트)
```

**금지:** 게시글 ID 매핑 · 첨부 URL 공유 · 통합 검색 인덱스 · 크로스 embed

## 8. 자료실 상세 기획 (library boardKey)

### 8-1. 콘텐츠 유형

| 유형 | 예 | 메타 |
|------|-----|------|
| PDF 가이드 | 안전과외 요약 | 역할, 태그 |
| 양식 | 수업 계약 체크리스트 | 다운로드 수 |
| 운영 안내 | 공부방 등록 스크린샷 | 링크 연결 14장 |
| FAQ PDF | 인쇄용 FAQ | 고객센터 연동 |

### 8-2. UI (디자인 일괄 전 — 패턴만)

- **목록:** 카테고리 탭 + 카드(제목·요약·파일형·날짜)
- **상세:** 본문 + 다운로드 (Study114 스토리지만)
- **고객센터:** 동일 사이트 내 라우트 링크만 — **커뮤니티 URL 없음**

### 8-3. 커뮤니티와의 관계

**없음.** 자료실 파일·메타는 Study114에만 보관. 커뮤니티 첨부와 **공유·동기화·노출하지 않는다.**
