## 0. 한 줄 요약

> Study114 안에는 공지·FAQ·가이드·양식·다운로드·제출·공유를 담는 **게시판 엔진**을 둔다.

## 2. 회원 연동 — 로그인·신원만 (콘텐츠·역할 상세 비공유)

### 2-1. 정본·파생

| 시스템 | 회원 테이블 | 정본 여부 |
|--------|-------------|-----------|
| Study114 | `users` + `user_profiles` | **정본 (Master)** |
| 그누보드 | `g5_member` | **파생 (Linked)** — 커뮤니티 전용 |

### 2-2. 동기화 필드 (로그인·신원만)

| 필드 | 방향 | 비고 |
|------|------|------|
| email (ID) | Study114 → GNU | 로그인 키 |
| name | S114 우선 | 표시명 |
| password | **동기화 안 함** | SSO만 |
| status | S114 → GNU | 활성/차단/탈퇴 |
| phone, gender, birth | 선택 | 기초 신원 |

**동기화하지 않을 것 (명시):**

- 게시글·댓글·첨부·자료실 파일
- 공부방/과외/학생 프로필·등록 데이터
- 쪽지·찜·비교·노출(Prime/Pick) 상태
- Study114 **역할별 세부 권한** (커뮤니티에 그대로 넘기지 않음)

### 2-3. 커뮤니티 권한 — 단순 2단 (후순위·마케팅)

Study114 내부는 역할·유료등록이 복잡해도, **커뮤니티에서는 단순화**:

| S114 매핑 (로직 예시) | GNU `mb_level` | 커뮤니티 권리 |
|----------------------|----------------|---------------|
| 비로그인 | — | 읽기만 또는 가입 유도 |
| 로그인 (무료) | `1` 무료회원 | 글쓰기·댓글·첨부 (동일) |
| 유료 (공급자 유료등록 등 조건) | `2` 유료회원 | **무료와 동일 권리** + 뱃지·노출만 차별 가능 |

> **의도:** 커뮤니티에서 **「유료도 무료도 같은 권한」** — 기득권·마케팅 («본서비스 유료 = 커뮤니티도 우대 체감»)  
> 실제 차별은 포인트·뱃지·프로필 장식 등 **가벼운 요소**로, 쓰기 권한 자체는 맞춤.

### 2-4. 연동 테이블 (Study114 측)

```
member_external_links
  - id
  - user_id          → users.id
  - provider         → 'gnu_board5'
  - external_id      → g5_member.mb_id
  - last_synced_at
  - sync_status
```

### 2-5. 로그인 흐름 (목표)

```
[사용자] Study114 로그인 (auth)
    → 세션/JWT 발급
    → 커뮤니티 진입 시 signed token 또는 SSO endpoint
    → 그누보드 custom auth / 연동 플러그인
    → g5_member 자동 생성·갱신 (기초정보만)
```

| 방식 | 장점 | 단점 |
|------|------|------|
| **A. SSO 토큰 (권장)** | 비밀번호 이중관리 없음 | 그누보드 커스텀 개발 |
| B. 비밀번호 미러링 | 구현 단순 | 보안·탈퇴 동기화 지옥 |
| C. 완전 분리 계정 | 독립 | UX 나쁨 · 연동 목적 상실 |

**1차:** Study114 로그인 → 커뮤니티 진입 시 **자동 로그인(SSO)**  
**UI:** 본서비스 GNB에 「커뮤니티」= **외부 사이트 링크** (임베드·iframe·글 API 없음)

## 3. Board Engine — 하나의 엔진, 여러 채널

### 3-0. 게시판 분류 체계

| 구분 | 설명 | 예 |
|------|------|-----|
| **운영형** | 운영자가 공지·안내·정책·가이드 발행 | notice, faq, safe-guide |
| **다운로드형** | PDF·양식·체크리스트 · 썸네일·다운로드 | library 계열 |
| **권한형 업로드** | 특정 역할 write/upload · 제출·사례 | submission (1차 모델) |
| **큐레이션형** | 운영 선별 노출 | showcase (2차 후보) |
| **외부 커뮤니티** | GNU · 별도 DB | community.* |

### 3-1. 권한 모델 (boardKey별)

**권한 축:** `read` · `download` · `write` · `comment` · `upload` · `edit_own` · `delete_own` · `moderate`

**역할 축:** `guest` · `member` · `demand`(학부모) · `supply-room` · `supply-tutor` · `verified` · `admin`

**보드 메타 (최소):** `visibility` · `allowedRoles` · `allowComment` · `allowUpload` · `requireReview` · `downloadPolicy`

코드 정본: `board-engine-copy.js` · `BOARD_REGISTRY`

### 3-2. boardKey 목록 (권장안)

| boardKey | 용도 | 노출 위치 | 성격 | 기본 권한 |
|----------|------|-----------|------|-----------|
| `notice` | 공지 | 17장 고객센터 | 운영형 | admin write / 전체 read |
| `faq` | FAQ | 17장 고객센터 | 운영형 | admin write / 전체 read |
| `safe-guide` | 안전과외 | 17장 고객센터 | 운영형 | admin write / 전체 read |
| `policy-log` | 약관 변경 이력 | 26장·푸터 | 운영형 | admin write / 전체 read |
| **`library`** | **자료·다운로드** | **`#/library` (유틸)** | **다운로드형** | admin write / 로그인 read·download |
| `library-template` | 양식·체크리스트 | 자료실 하위 | 다운로드형 | admin write / 로그인 download |
| `library-guide-pdf` | 가이드 PDF | 자료실·고객센터 | 다운로드형 | admin write / 공개 read · 로그인 download |
| **`submission`** | **제출·업로드** | 후순위 route | **권한형 업로드** | 특정 역할 write/upload · 검토 후 노출 |
| `showcase` | 사례 공유 | 2차 후보 | 큐레이션형 | 공급자 write · 운영 검토 |

### 3-3. 「자료실」과 게시판 엔진의 관계

- **게시판 엔진** = 23장 상위 개념
- **자료실** = 다운로드형 boardKey(`library` 계열)에 대한 **사용자-facing 명칭 후보**
- `/library` route 가 남아 있어도 23장 전체를 「자료실 장」으로 부르지 않는다
- 30장 부록 C도 동일 해석 — `library` = 엔진 하위 채널

### 3-4. 고객센터 vs 자료실 — 같은 엔진, 다른 규칙

| 구분 | 고객센터 보드 (운영형) | 자료실 (`library` 계열 · 다운로드형) |
|------|------------------------|-------------------------------------|
| 목적 | 신뢰·안내·정책 | 학습·운영 **다운로드·참고** |
| 작성 주체 | 운영자 (`admin`) | 운영자 (후순위: 권한형 업로드 보드는 역할별) |
| UI 패턴 | 아코디언·게시판형 | **카드/리스트 + 파일형 + 다운로드** |
| 권한 | 전체 read · admin write | boardKey별 read/download (로그인·역할) |
| 댓글 | 없음 (1차) | 없음 (1차) |

### 3-5. 업로드형 게시판 정책 (핵심 추가)

| 항목 | 원칙 |
|------|------|
| 작성 권한 | 전체 공개 ✕ · 역할/상태 기반 |
| 파일 제한 | 형식·용량·개수 제한 필요 |
| 노출 | 즉시 공개 vs `requireReview` — 보드별 |
| 운영 조치 | 숨김·고정·비공개·신고 (A28) |
| GNU와 구분 | 자유 UGC = GNU · 제출/사례 = Study114 권한형 |

### 3-6. 자료실 메뉴 (6장과의 관계)

- 6장: GNB **「자료실/포털 대확장」 1차 제외**
- **급한 데로:** 유틸 메뉴 또는 고객센터 인근 **「자료실」** 1개 → `library` 보드
- **대확장(후순위):** 카테고리·검색·인기자료·역할별 추천

```
[유틸] 자료실  →  #/library  (home-ui 또는 전용 라우트)
                    → Board Engine library 보드
                    → 파일 다운로드 (CDN/스토리지)

[마이페이지] 제출함  →  #/mypage/submission-board  (P23-04)
                    → boardKey `submission` · 권한형 업로드
                    → P15-10 제출자료 상태와 브리지
```

## 6. 데이터·API 경계 (콘텐츠 격리)

```
[Study114] — 자체 닫힌 영역
  /api/board/...          ← notice, faq, library (GNU API 호출 없음)

  /bbs/...                ← 커뮤니티 only

[유일한 공통 통로]
  /api/sso/issue-token    ← S114 로그인 후 커뮤니티용 짧은 토큰

  member tier sync        ← 무료|유료 → mb_level (배치/이벤트)
```

**금지:** 게시글 ID 매핑 · 첨부 URL 공유 · 통합 검색 인덱스 · 크로스 embed

## 8. 자료실 상세 기획 (library boardKey)

### 8-1. 콘텐츠 유형

| 유형 | 예 | 메타 |
|------|-----|------|
| PDF 가이드 | 안전과외 요약 | 역할, 태그 |
| 양식 | 수업 계약 체크리스트 | 다운로드 수 |

### 8-2. UI (디자인 일괄 전 — 패턴만)

- **목록:** 카테고리 탭 + 카드(제목·요약·파일형·날짜)
- **상세:** 본문 + 다운로드 (Study114 스토리지만)
- **고객센터:** 동일 사이트 내 라우트 링크만 — **커뮤니티 URL 없음**

### 8-3. 커뮤니티와의 관계

**없음.** 자료실 파일·메타는 Study114에만 보관. 커뮤니티 첨부와 **공유·동기화·노출하지 않는다.**
