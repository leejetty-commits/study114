### 2번 상세 — 비로그인 티저 B-완화형

**원칙:** 조건은 보이되 사람은 특정되지 않게. “있다”는 감만 · 판단 재료는 과하게 주지 않음.

**항상 마스킹 (로그인/비로그인):** 공개 표시명 — 예: `김○○`, `이○`, `박○학생`

**비로그인에서 보여도 됨**

| 항목 | 예 |
|------|-----|
| 마스킹된 공개 표시명 | 김○○ |
| 희망 유형 | 공부방 희망 / 과외 희망 |
| 학교급만 | 초등 · 중등 · 고등 (중2·고1 등 정확 학년 ✕) |
| 대표 희망 과목 **1개** | 수학 |
| 희망 지역 거친 단위 | 대치동 · 잠실권 (단지명 ✕) |
| 예산 구간형 | 40~60만원 |
| 수업형태/인원 중 **1개** | 1:1 희망 · 그룹 희망 · 주 2회 희망 |

**비로그인에서 가림 (블라인드)**  
실명 원형 · 정확 학년 · 성별 · 정확 주소 · 단지명 · 학교명 · 요청문 본문 · 특이요청 · 연락 가능 시간 · 희망 강의스타일 전체 · 세부 일정 · 정확 예산 · 형제자매/가정상황 추정 가능 자유서술

**카드 예시**

- `김○○ · 중등 · 수학 · 대치동 · 40~60만원`
- `박○학생 · 고등 · 영어 · 잠실권 · 1:1 희망`

**구현:** `student-blind-teaser.js` · 게스트 `renderBasicStudentRow` 티저 카드

### 3번 상세 — 학부모·학생 학생찾기 비교 열람

**논리:** 공부방 로그인 시 다른 공부방을 보듯, 학생 로그인도 다른 학생의 등록 조건(금액대·지역·과목·수업형태·예산 수준)을 비교 열람한다. 시장 참여자로서의 비교를 허용.

**열람 허용 ≠ 블라인드 해제**

| 유지 | 내용 |
|------|------|
| 표시명 | 로그인 무관 항상 일부 마스킹 |
| 실명·연락처·정확 주소 | 비노출 |
| 자유서술 민감정보 | 비노출 |
| 요청문·특이요청 | 기존 공개 규칙 유지 (피어는 본문 비공개 · 공급자 유료/열람권 규칙) |
| 쪽지 | 학부모→다른 학생 콜드 쪽지 불가 |

**노출 방식:** 공급자와 동일한 블라인드 리스트(구조화 조건 카드) · GNB「학생찾기」·`#/parent`「우리동네 학생」탭

### 12번 상세 — 과외쌤 Prime vs 공부방 Prime

| | 공부방 Prime | 과외쌤 Prime |
|--|-------------|-------------|
| 지역 단위 | 동(기존) | **시** |
| 슬롯 | 3 고정 | 화면당 3 · **페이지네이션** |
| 시간 순환 | 없음 | **약 15분** 세트 회전 (`pick_rotation_minutes`) |
| EMPTY | 미점유 슬롯 홍보카드 | 후보 없을 때만 3칸 EMPTY |
| overflow | 정책 확정 전 보류 | 페이지로 소화 |

데모 후보 풀: `demo_prime_tutor_pool` (기본 12). Pick/Basic 제외는 후보 풀 전체 ID.
