# 가입 방식 (1차)

## 기본 방향

- **가입 주체**: 학부모(보호자) 기준 단일 회원 타입.
- **학생**: 별도 로그인 계정 없이, 회원 가입 후 **자녀 정보**를 추가·관리.
- **만 14세 미만**: 1차에서 별도 분기·법정대리인 동의 플로우 **미구현** (추후 확장 포인트만 문서화).

## 가입 플로우 (1차)

```
[회원가입]
  1. 이메일 + 비밀번호 (+ 비밀번호 확인)
  2. 학부모 필수: 이름, 휴대폰 (인증은 2차 검토)
  3. 선택: 거주 지역 (시/구/동 또는 단지)
  4. 약관 동의 (서비스 이용, 개인정보)
  5. 가입 완료 → 기본 role: member (일반 회원)

[로그인]
  - 이메일 + 비밀번호
  - 세션 또는 JWT (구현 단계에서 결정, 1차는 세션 권장)

[자녀(학생) 등록] — 마이페이지
  - 이름, 학년(또는 학교급), 선택: 학교명
  - 1차 최소 필드만

[역할 부여] — 등록 시 자동
  - 공부방 최초 등록 → user_roles에 study_room_owner 추가
  - 과외쌤 최초 등록 → user_roles에 tutor 추가
  - 동일 계정에 두 역할 공존 가능
```

## 회원 상태

| status | 설명 |
|--------|------|
| `active` | 정상 |
| `inactive` | 휴면·비활성 (관리자) |
| `withdrawn` | 탈퇴 (soft delete, 이메일 재사용 정책은 운영 시 결정) |

## 인증·보안 (구현 가이드)

- 비밀번호: `password_hash()` / `PASSWORD_DEFAULT` (bcrypt)
- 이메일 unique
- CSRF 토큰 (폼 POST)
- 1차: 이메일 인증·휴대폰 OTP **선택** (스키마만 nullable 컬럼 확보)

## DB 매핑

| 개념 | 테이블 |
|------|--------|
| 계정 | `users` |
| 학부모 프로필 | `user_profiles` |
| 자녀 | `students` |
| 역할 | `user_roles` |

## 추후 확장 (1차 미포함)

- 만 14세 미만 자녀 전용 동의·본인인증
- 소셜 로그인 (카카오, 네이버)
- 학생 독립 계정·학부모 연동
- 사업자(공부방 원장) vs 개인 과외쌤 구분 KYC
