## 2. 공통 상태 카드 템플릿 (5요소)

| 요소 | 설명 |
|------|------|
| **제목** | 현재 상태 한 줄 |
| **보조 설명** | 왜 이런 상태인지 짧게 |
| **CTA** | 지금 할 수 있는 다음 행동 |
| **보조 링크** | 로그인 · 검색 · 등록 · 유료안내 등 |
| **톤** | 탓하지 않음 · 압박 없음 · 차분한 안내 |

### 작성 순서

1. 무엇이 비어 있거나 제한되어 있는지
2. 왜 그런 상태인지 짧게
3. 다음 행동
4. (필요 시) 보호/정책 안내

## 3. 권한 부족 5종 (뭉뚱그리지 않음)

| 유형 | 예시 문구 |
|------|-----------|
| **비로그인** | 로그인 후 자세한 정보를 볼 수 있습니다. |
| **역할 제한** | 이 기능은 과외쌤 역할에서 이용할 수 있습니다. |
| **유료 필요** | 이 기능은 유료등록시 가능합니다. |
| **학생 보호 제한** | 학생 정보는 보호를 위해 제한적으로만 공개됩니다. |
| **비공개 정보** | 이 정보는 아직 공개되지 않았습니다. |

권한 부족 / 보호 제한 / 비공개를 **같은 말로 섞지 않는다.**

## 4. Error 4종

| 상황 | 문구 | CTA |
|------|------|-----|
| 데이터 로드 실패 | 불러오지 못했습니다. 잠시 후 다시 시도해 주세요. | 다시 시도 |
| 저장 실패 | 저장하지 못했습니다. 입력한 내용을 확인한 뒤 다시 시도해 주세요. | 다시 저장 |
| 상태 변경 실패 | 상태를 변경하지 못했습니다. 잠시 후 다시 시도해 주세요. | 다시 시도 |
| 일시적 서버 오류 | 일시적인 문제가 발생했습니다. 잠시 후 다시 이용해 주세요. | 새로고침 |

**원칙:** 원인 단정 ✕ · 저장 실패 시 **입력값 유지** 우선

## 5. Empty / 0건 / Max 구분

| 상태 | 의미 | 예 |
|------|------|-----|
| **Empty** | 사용자 데이터가 **아직 없음** | 찜 없음 · 최근열람 없음 · 쪽지 0건 |
| **0건** | **검색**했지만 결과 없음 | 조건에 맞는 결과가 없습니다. 지역이나 조건을 넓혀보세요. |
| **Max** | **상한** 도달 | 비교 후보는 최대 3개까지 담을 수 있습니다. |

---

## 6. 학생 보호 = “보호 안내” UX

학생 관련 제한은 **금지/거절**이 아니라 **보호 원칙**으로 설명한다.

- ✕ 권한이 없습니다
- ○ 학생 정보는 보호를 위해 제한적으로만 공개됩니다.
- ○ 학생 상세 정보는 공개 범위 안에서만 확인할 수 있습니다.
