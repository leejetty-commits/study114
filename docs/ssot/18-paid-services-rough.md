## ★ 심도 있게 주목하는 포인트 (정책 핵심)

설계·구현·카피에서 **아래를 최우선**으로 본다. SKU나 과금 메커니즘이 아무리 잘 짜여도, 이 전제가 깨지면 공급자는 지갑을 열지 않는다.

| # | 포인트 | 의미 |
|---|--------|------|
| **1** | **가게 꾸미기 = 무료** | 상세등록·비교 친화 프로필·Basic 노출은 **유료가 아님** (5장 정보/광고 분리 · 14§6) |
| **2** | **유료 = 홍보·획득** | 잘 꾸며진 가게를 **더 보이게 · 더 연락하게** — 전단지·동네 경쟁 대체 채널 |
| **3** | **free = 대기 중인 샵** | 학부모→공급자 **선연락은 free 허용** · 공급자→학생 **콜드 메모만 유료** (16§8) |
| **4** | **Prime ≠ 포인트** | Prime = **희소 슬롯** · Pick/Hot/점프/메모 = **포인트 소비형** — 정체성 혼동 금지 |
| **5** | **ROI 3종은 무료** | 조회·찜·비교 담김 — **효과를 보고** 과금 · 인사이트·알림만 후순위 유료 |
| **6** | **동네 단위·저숫자 OK** | 광역 플랫폼이 아님 — **실제 연결 가능한 동네 수요**의 관심이면 수치가 낮아도 의미 있음 |
| **7** | **신뢰 vs 주목 배지 분리** | 교육청·검증 = **비매·심사** · Hot/Prime = **유료** — **UI 색·위치·아이콘**까지 분리 (11·14장) |
| **8** | **공급자 UI는 단순** | 내부는 하이브리드·포인트·카탈로그 — **겉은** 샵 완성도 / 반응 보기 / 더 노출·연락 **3층** |
| **9** | **첫 상품 경험 ≠ 월 정액** | 내부는 하이브리드여도 **대외 첫 과금**은 **무료 샵 + 단기 부스트** — 월 정액은 효과 체감 **후** |

**시장 맥락 (공부방):** 출혈경쟁·신규 원장의 홍보 절박·학부모의 비교·분석형 결정 — Study114는 **「비교에 이기는 샵」+「측정 가능한 동네 노출」**을 먼저 증명한 뒤 과금한다.

**헤드라인 (대외·문서 최상위):** **가게 품질 무료 · 홍보·획득 유료**

## 0. 핵심 문장 (잠금)

**전제**  
Study114 유료는 **「가게 꾸미기」가 아니라 「잘 꾸며진 가게의 홍보·획득」**에 대한 과금이다. **가게 꾸미기 = 무료**가 선행 조건이며, 이것이 만족될 때 비로소 과금이 유의미하고 공급자가 지갑을 연다.

**과금**  
공급자는 **월 정액**으로 기본 홍보 entitlement를 확보하고, **메모·요청문 열람·Pick·Hot·점프** 같은 **소비형** 기능은 **포인트**로 추가 구매한다.

**Prime 예외**  
**Prime**은 포인트 상품이 아니라 **희소 슬롯형** 운영 자산으로 분리한다. (`paid` = Prime **신청 자격** · Prime 3칸 = **운영 배정**)

**접촉**  
**학부모→공급자** 선연락은 **free 허용** · **공급자→학생** 메모(콜드 아웃리치)만 **유료** — 「먼저 관심 보인 학부모에게 답하기」vs「먼저 찔러보기」구분.

**ROI**  
조회·찜·비교 담김 = **기본 무료 노출** · 지역 수요 추이·동등급 대비·알림형 인사이트 = **후순위 유료**.

**한 줄 요약**

- **구매 주체:** 공급자만 · 학부모/학생 **무과금** (15§7 · 17§FAQ)
- **하지 않는 것:** 에스크로 · 지급 중개 · 학부모 유료 UI · **신뢰 배지 유료화**

## 1. 수익모델 큰 축 (정리)

알바몬형 **4축**을 Study114 맥락에 맞게 패키징하고, 2026식으로 **포인트·리드·ROI·배지 분리**를 얹는다.

### 1-1. 알바몬형 4축 (기본 DNA)

| 축 | Study114 대응 | free / paid |
|----|---------------|-------------|
| **① 등급·기간 과금** | `free` / `paid` · 월 정액 · 기간권(30/90일 등) | paid = 기본 entitlement |
| **② 잠긴 기능 횟수 개방** | 요청문 열람 · 공급자→학생 메모 · (포인트 소비) | free 제한 · paid/포인트 확장 |
| **③ 배지·주목도 과금** | Prime / Pick / Hot / 점프 · **주목 배지** | 신뢰 배지와 **분리** |
| **④ 광고·배너** | 홈 우측 · 리스트 스폰서드 · (후순위) 교육 인접 직판 | 초기 **비주력** · ADX ✕ |

### 1-2. 2026식 업그레이드 (4축 위에 얹음)

| 레이어 | 내용 |
|--------|------|
| **포인트 지갑 (통합)** | 공부방·과외 **역할 통합** 잔액 · **소비형 SKU만** 포인트 결제 |
| **행동/리드 단위** | 노출만이 아니라 **메모 1회 · 요청문 열람 · 부스트 N일** |
| **ROI 미니 대시보드** | P15-09 · P18-02 — **무료 3종** + (후순위) 인사이트 |
| **시즌 캠페인 태그** | 중간고사 7일 · 방학 14일 — **새 SKU 체계 ✕** · Pick/Hot/점프에 **태그** |

### 1-3. 과금 유형 4분류 (포인트 범위 잠금)

| 유형 | 예 | 과금 수단 | 포인트? |
|------|-----|-----------|:-------:|
| **자격형** | `paid`, Prime 신청 가능 | 월 정액 / entitlement | **✕** |
| **희소형** | Prime 3칸 슬롯 | 운영 배정 · 심사·기간 | **✕** |
| **소비형** | 메모, 요청문, Pick, Hot, 점프, 시즌 태그 | **포인트** (월 포함분 + 충전) | **○** |
| **후순위형** | 인사이트, 알림 우선 | 별도 SKU 또는 포인트 | △ 18d+ |

> **경계:** 자격형·희소형을 포인트에 넣으면 체계가 다시 흐려진다. **Prime 대기권 ≠ 포인트 SKU.**

### 1-4. 공급자가 듣는 말 (대외 3층)

내부 카탈로그와 별도로, 원장·과외쌤 화면은 **최대 3층**만 노출한다.

1. **내 샵** — 완성도 · 미리보기 · 학부모 비교 화면 (**무료**)
2. **반응** — 조회 · 찜 · 비교 담김 (**무료**)
3. **더 보이기 / 연락** — Pick·Hot·메모 등 (**유료·포인트**)

Pick / Prime / Hot / 포인트 **용어는 P18-01·FAQ** — 일상 UI에는 「3일 우리 동네 상단」 등 **효과·기간** 중심.

## 6. ROI 정책

### 6-1. 무료 가시성 (P15-09 · P18-02)

| 지표 | 노출 | 비고 |
|------|:----:|------|
| 조회 | ○ | |
| 찜 | ○ | |
| 비교 담김 | ○ | 11장 비교 UX |
| 메모 **시도/차단** | **✕** | 내부 업셀만 |

### 6-2. 동네·저숫자 원칙

- Study114는 **광역 매칭이 아님** — 저장 지역·동·학교권 단위.
- **실제 등원 가능한 동네 수요자**의 관심(찜·비교·문의)이면 **절대값이 낮아도** 유의미.
- KPI·카피에서 **전국 플랫폼식 높은 숫자**를 기대하게 하지 않는다.
- 유료 전환 논거: 「숫자가 많아서」가 아니라 **「우리 동네에서 비교 중인 학부모에게 보였다」**.

### 6-3. 후순위 유료 인사이트

- 지역·과목 **수요 추이**
- 동등급·인근 **대비 성과**
- 비교·찜 **추세**
- 알림형 상품 — **16장 메시지 정책**과 함께 18d+ (1차 SKU ✕)

### 7-2. Prime / Pick / entitlement

```
free  → Basic + 샵 완성 + ROI 3종 + 학부모 선연락
paid  → + Prime 신청 자격 + 월 포함 소비 할당 + 포인트 충전
Prime 3칸 → paid 중 운영 배정 (포인트 구매 ✕)
Pick/Hot/점프 → 포인트 소비 (시즌 태그 optional)
```

11장 **「Prime/Pick도 Basic 리스트에 중복 가능」** 유지.

### 7-3. 시즌 캠페인 (태그)

- **새 상품 체계 추가 ✕** — 기존 Pick / Hot / 점프에 `campaign: midterm_7d` 등 **운영 태그**
- 예: 중간고사 7일 · 방학 14일 · 신학기 부스트

| **월 정액** | `paid` + 기본 entitlement + **월 포인트/기본 할당** (수치 → [18b](18b-paid-catalog-placeholder.md)) |

| **지갑** | **통합** (공부방·과외 동일 user) |
| **포인트 적용 범위** | **소비형만** — 자격·Prime 희소 **제외** |

### 8-1. 내부 vs 대외 (잠금)

| 층 | 내용 |
|----|------|
| **내부 설계** | 월 정액 + 포인트 **하이브리드** |
| **대외 첫 상품 경험** | **무료 샵 완성** → **ROI 3종 확인** → **단기 부스트**(3·7일 Pick/Hot) |
| **월 정액 위치** | P18-01·결제 UI의 **전면 1순위 ✕** — 효과 체감·반복 구매 **후** |
| **free 생존** | 학부모 **선연락 수신·답장** · ROI 3종 — **항상 살아 있음** (16§1-2) |

**신규 공급자 진입:** 90일 정액보다 **3·7일 부스트** 우선 (전단지 대체 심리).

## 15. 회의 YES/NO — 잠금 6항

1. **과금:** 하이브리드 · 포인트는 **소비형만** · **통합 지갑**
2. **카탈로그:** Prime=희소 / Pick·Hot·점프·시즌=소비 / 메모·요청문=열쇠
3. **배지:** 신뢰=비매 / 주목=유료 / 매칭=노출 개수 · **UI 분리**
4. **ROI:** 조회·찜·비교 담김=무료 / 인사이트·알림=후순위
5. **가치 선행:** **가게 꾸미기=무료** · 유료=홍보·획득
6. **접촉:** 학부모→공급자 **free** · 공급자→학생 콜드 **유료**
7. **대외 첫 과금:** **단기 부스트** · 월 정액 전면 1순위 **✕**

## 17. 오픈 이슈 (18b 이후 · 수치·운영)

1. free **요청문 열람** 월 맛보기 횟수 (0 vs 1~3)
2. Prime 배정: 선착순 vs 심사 vs (후순위) 입찰
3. 공부방 vs 과외 **카탈로그 카피** 분리 수준 (지갑은 통합)
4. 환불 시 포인트·노출 기간·entitlement 회수
5. `paid` ↔ 23장 `mb_level` **동일 조건** 문장 확정
6. 스폰서드 vs 공급자 자기홍보 **슬롯 규칙**
7. 알림형 상품 — 16장 정책 후

**닫힌 이슈 (재논의 ✕):** Prime≠포인트 · ROI 3종 무료 · 신뢰/주목 분리 · 학부모 무과금 · 선연락 free · **헤드라인=가게 품질 무료·홍보획득 유료** · **첫 과금=단기 부스트**
