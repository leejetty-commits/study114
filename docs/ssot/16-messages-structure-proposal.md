## 1. 목적

16장은 **「쪽지 기능 설명서」가 아니다.**  
13장 §8에서 잠근 권한이 **실제 화면에서 어떻게 보이고 · 막히고 · 통과되는지**를 정의하는 장이다.

| 원칙 | 내용 |
|------|------|
| 공식 접촉 | 전화·메일 **플랫폼 비노출**(6장) |
| 블라인드 | 실명·연락처 중계 없음 · **표시명** 기준 |
| 권한 연동 | 13장 §8 · **접촉 방향**(§1-2) — 콜드 메모만 유료 · `paid_only` visibility |
| **정본 UI 약속** | 「대화가 되느냐」보다 **「이 대화 안에서 무엇까지 공개되느냐」** — 공개 범위 배지(P16-02) |

**15장과의 관계:** P15-08 = **요약·진입(허브)** · P16-xx = **쪽지함 본문**.

### 1-1. 정본 4축 (우선순위)

| 순위 | 축 | 화면 ID | 역할 |
|:----:|----|---------|------|
| 1 | **권한표** | §8 | 역할·유무료별 허용/차단 |
| 2 | **공개 범위 배지** | P16-02 | 13장 §8의 사용자-facing 표현 |
| 3 | **첫 메모 플로우** | P16-03 | 22장·검색·상세 → thread 생성 |
| 4 | **유료 게이트** | P16-04 | 무료 공급자 차단 UX → 18장 |

검색·보관함·고급 메시지 기능은 **후순위**(§10).

### 1-2. 접촉 방향 정본 (18장 §0 · §4 연동)

> **한 줄 (잠금):** **학부모→공급자 선연락·답장은 free** · **공급자→학생 콜드 메모만 유료**(P16-04).

| 방향 | 주체 | free | 횟수권/기간형 | 게이트 |
|------|------|:----:|:-----------:|:------:|
| **선연락** | 학부모 → 공급자 | ○ | ○ | 없음 |
| **답장** | 공급자 → 학부모 (선연락 thread) | ○ | ○ | 없음 |
| **콜드** | 공급자 → 학생 (먼저 메모) | ✕ | ○ | **P16-04** |

- free 공급자 = **대기 중인 샵** — 받은 쪽지·학부모 문의에 **답장 가능** · 학생찾기에서 **먼저** 메모만 차단.
- P16-04는 **콜드 아웃리치**에만 적용 — 학부모 화면·선연락 수신 **해당 없음**.

### 1-3. 역할별 접촉 채널 비중 `[원본 · 2026-07-06]`

쪽지는 우동공과의 **공식 접촉 채널**이다.  
다만 **역할별 사용 빈도와 운영 UI 비중은 다르다.** 권한 모델(§8)은 **공통** · 화면·CTA **강조는 역할별**.

| 역할 | 쪽지·상담 빈도 | 운영 UI 해석 |
|------|----------------|--------------|
| **공부방** | **저** — 결정 직전 **보조** 채널 | [20장](20-study-room-registration-management.md) `inquiry_status` = **상담 수용 표지판** (대화량·활성도 ✕) |
| **과외쌤** | **高** — 학생찾기·조건 협의·콜드 메모 | [21장](21-tutor-registration-management.md) **학생 접근·쪽지** = **1급 운영축** |
| **학부모** | 공급자와의 공식 접촉 | **무료 중심** · 16장 본문 |

> **공통 (변경 없음):** 학부모→공급자 선연락·답장 **free** · 공급자→학생 **콜드 메모** 유료 — §1-2 · [18§4](18-paid-services-rough.md#4-접촉메모-정책-168-연동--잠금).

**참조:** [15§4-3](15-mypage-structure.md#4-3-숫자-요약--cta) 홈 강조 · [18§4-1](18-paid-services-rough.md#4-1-역할별-유료-안내-순서) SKU 순서 · [20§4-3](20-study-room-registration-management.md#4-3-축-b--상담-수용-상태-inquiry_status-1차-ddl--api) · [21§4-3](21-tutor-registration-management.md#4-3-매칭-가시성-ui-계산--20장-inquiry_status-대칭) · [22§9](22-platform-lifecycle-principles.md#9-역할별-접촉-채널-비중-요약) 요약.

## 3. 공개 범위 배지 (P16-02 · 정본 UI)

사용자에게 보이는 **16장의 핵심 약속**이다. 대화방 상단(P16-02)에 **항상 노출**한다.

| 수신자·상대 조합 | 배지 문구 | 요청문/특이요청 |
|------------------|-----------|----------------|
| 무료 공급자 → 학생 (콜드) | `구조화 항목만` | **비노출** · 선발송 **차단 → P16-04** |
| 유료 공급자 → 학생 | `구조화 + paid_only` | visibility=`paid_only`만 요약 패널·인용 |
| 학부모 → 공급자 | `공개 프로필` | 공급자 상세 공개 범위(11·22장) · **게이트 없음** |
| 공급자 → 학부모 (답장) | `공개 프로필` | **선연락 thread 내 답장 · free 허용** |

리스트 행(P16-01)에도 **동일 칩**을 축약 표시한다.

## 4. P16-01 쪽지함 리스트

```
┌─────────────────────────────────────────┐
│ [받은] [보낸] [진행중]                    │  ← 1차 탭 (검색 없음)
├─────────────────────────────────────────┤
│ ● 김수학(과외) · 수학 · 2시간 전          │  ← unread dot
│   "안녕하세요, 주 2회 가능합니다…"        │
│   [학생 의뢰] [구조화 정보만]             │  ← 컨텍스트 · 범위 칩
├─────────────────────────────────────────┤
│   대치 우등생 공부방 · 1일 전             │
│   …                                      │
└─────────────────────────────────────────┘
```

> **검색창:** 와이어 예시에 넣지 않음. 필요 시 2차 — §10.

**행 필드**

| UI | 데이터 |
|----|--------|
| 상대 표시명 | tutor_display_name / study_room_name / public_display_name |
| 역할 아이콘 | 공급자 / 학부모 |
| 미리보기 1줄 | 최신 message body (마스킹) |
| 시각 | updated_at |
| 컨텍스트 | 어떤 상세/검색에서 시작했는지 `[임시]` |
| 범위 칩 | §3 배지 축약 |

**빈 상태**

| 역할 | 문구 |
|------|------|
| 학부모 | 「관심 있는 공부방·과외쌤에게 메모를 보내 보세요」 |
| 공급자(무료) | 「받은 문의에 답장해 보세요」·「학생에게 **먼저** 메모는 유료등록 후」→ P16-04 |
| 공급자(유료) | 「학생찾기에서 의뢰를 확인하고 메모를 보내 보세요」 |

```
┌─────────────────────────────────────────┐
│ ← 목록    맑은하늘(학생)    [신고][보관]  │
```

**상단 · 우측 액션**

| 액션 | 1차 | 층 |
|------|:---:|-----|

| 상대 기본정보 요약 | ○ | 패널 |

| 차단/종료 | △ | 상태·오버레이 |

**차단/제한 상태:** 입력·전송 disabled · 사유 1줄 — **별도 페이지 아님**.

## 6. P16-03 첫 메모 보내기

### 6-1. 진입점 (트리거)

| 출처 | 주체 | 1차 |
|------|------|:---:|
| 학생 상세(22장) | 공급자 | ○ |
| 과외/공부방 상세 | 학부모 | ○ |
| 13장 검색 · 메모 버튼 | 공급자 | ○ |
| P15-06 찜 | 학부모 | △ |
| P15-07 최근열람 | 공통 | △ |

### 6-2. 플로우

```
[메모 보내기 클릭]
  → 접촉 방향 판정 (§1-2)
      → 학부모→공급자 · 공급자→학부모(답장): 권한 검사 통과 → P16-03
      → 공급자→학생(콜드): 권한 검사 (13장 §8)
          → 실패: P16-04 게이트 (오버레이)
          → 성공: P16-03 모달
              → 본문 입력 (템플릿 선택 `[임시]`)
              → 전송
              → P16-02 (기존 thread 또는 신규 — §6-3)
```

### 6-3. thread 재사용 `[임시]`

**동일 participant 쌍 + 동일 context**(student_id / tutor_id / study_room_id · 진입 출처)이면 **기존 thread를 재사용**한다.  
신규 thread 남발·스팸·법무 정책은 후순위에서 보완.

**모달 필드**

| 필드 | 필수 |
|------|:---:|
| 본문 | ○ |
| 연결 대상 (student_id / tutor_id / study_room_id) | hidden |
| 첫 인사 템플릿 | 선택 |
