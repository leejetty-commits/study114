## 0. 핵심 문장 (잠금 방향)

- **대상:** 과외쌤(`tutor`) · `tutors.user_id` 기준 **1 : 과외 프로필 N**
- **운영 단위:** 「등록 폼」이 아니라 **과외 카드** — 전환 → 저장 / 공개 / 수정 / 숨김 / 삭제
- **저장 ≠ 공개:** `profile_status` — `draft`는 검색·상세 **미노출** · `published`만 11·13장 노출
- **제출자료 ≠ 심사:** 학력·경력 등 **등록·공개 범위 선택** · 플랫폼 **표시만** · 운영자 승인·반려 **✕**
- **비교검색:** **있음** (6·11·13장)
- **신규 상태 enum:** 21장에서 **정의하지 않음** — 8·15·16·18장 기존 축 + **UI 계산값** 조합

**한 줄:** 21장 = **과외 카드 lifecycle + 학생 접근·쪽지 조종판 + 노출·부oost 조종판 + 4모드 미리보기 + 공개 전 자기점검** · 8장 + tutor-ui = **입력 필드 정의**.

> **내부 기획명:** Tutor Control Tower — 사용자-facing 명칭은 **과외쌤 운영센터**.

## 2. 목적

과외쌤은 학부모가 **비교·판단할 수 있는 조건형 서비스 프로필**(8장)이다. 등록 UI(tutor-ui)와 별도로 **저장 → 상세보강 → 미리보기 → 공개 → 숨김 → 학생 접근·노출 강화** 흐름을 한 세트로 잠근다.

**운영센터 컨셉:** 건강검진표 + **학생 접근·쪽지 조종판** + 노출·부oost 조종판 — 과외쌤이 「지금 공개 중인가 · 어떤 학생에게 접근 가능한가 · 무엇이 부족한가」에 즉답.

| 축 | 21장 | 타 장 |
|----|------|-------|
| lifecycle · `profile_status` | ○ | 8장 enum |
| **학생 접근·쪽지** | ○ | 16·18장 |
| **매칭 가시성** | ○ UI 계산 | 9·13장 |
| 제출자료 · P15-10 | ○ | 8장 · 15§2-1 |
| 과외 카드 · 전환 | ○ | tutor-ui |
| 기본/상세 **입력** | **✕** | tutor-ui |
| **상태판** · 3게이지 · 노출 매트릭스 | ○ | 11·18장 |
| **4모드** 미리보기 | ○ | 11·13장 |
| `inquiry_status` | **✕** | 20장 전용 |

### 2-1. 상태 enum 정책 `[잠금]`

> **21장은 신규 상태 enum을 정의하지 않고**, 8장 `profile_status` · `detail_completion_status`, P15-10 제출자료 상태, 16장 접근권, 18장 free/paid·포인트, 11·13장 노출 규칙을 **조합**해 운영 화면을 구성한다.

| `safety_review_status` enum | 공개 전 **자기점검** UI 계산 |
| `partial_public` · `suspended` 등 | 공개 준비 미완료 **계산값** |

## 4. lifecycle · 상태 (1축 DB + UI 계산)

### 4-1. 축 A — 공개 상태 `profile_status` (8장)

```
         ┌─────────┐
  저장   │  draft  │◄── tutor-ui 임시저장
         └────┬────┘
              │ P21-04 공개 (체크리스트 OK + 자기확인)
              ▼
         ┌───────────┐
   ┌────►│ published │◄────┐
   │    └─────┬─────┘     │  과외쌤 직접 전환 · 운영자 승인 없음
   │          │ 숨김       │
   │          ▼            │
   │    ┌──────────┐      │
   └────│  hidden  │──────┘
        └────┬─────┘
             │ soft delete
             ▼
        deleted_at
```

| `profile_status` | 화면 문구 | 검색·상세 | P21-01 탭 |
|------------------|-----------|:---------:|-----------|
| `draft` | 저장중 | ✕ | 저장 |
| `published` | 공개중 | ○ | 공개중 |
| `hidden` | 숨김 | ✕ | 숨김 |
| `pending` | **deprecated — 21장 UI 미사용** | — | — |

### 4-2. 공개 준비 · 3게이지 — UI 계산값 (DB 저장 ✕)

| 계산 상태 | 조건 | 용도 |
|-----------|------|------|
| **공개 가능** | 필수 체크리스트 충족 | P21-04 버튼 활성 |
| **공개 준비 미완료** | 필수 항목 부족 | P21-01 탭 · 상태판 |
| **상세 보강 권장** | 공개 가능하나 품질 항목 부족 | 한 줄 진단 |
| **노출·접근 강화 가능** | 상세 충분 + `published` | 18장 CTA |

**3게이지** — 퍼센트 **점수 ✕** · **N/M 항목** 또는 체크리스트:

| 게이지 | 의미 | 예시 항목 |
|--------|------|-----------|
| **프로필 완성도** | 입력 필드 충실 | 표시명 · 과목 · 활동 시 · 과외비 · 수업방식 · 소개 · 사진 |
| **신뢰정보 풍부도** | 학부모 판단 정보 **공개** 정도 | 학교·학과 · 학적 · 경력 · **제출자료 등록** · **공개 선택** |
| **노출·접근 준비도** | 검색·비교·접근 준비 | `published` · 주력과목 1 · 대표 활동 시 · 비교 필수값 · paid/메모 조건 |

> **신뢰정보 풍부도** = 플랫폼 **공식 신뢰 점수 ✕** · 「학부모가 판단할 정보가 얼마나 공개되어 있는가」.
