# 28장 — 관리자 최소 운영 콘솔 · RED LINE

**상태: 정책 2차 잠금** (2026-07-06 확정 · 대원칙·제출자료·혼용 금지 보강)  
**역할:** P17-admin과 A28 분리 · **우동공과를 심사·인증 플랫폼처럼 보이지 않게 하는 운영 금지선**  
**코드 정본:** `preview/home-ui/src/admin-red-line-copy.js`  
**연동:** [22장](22-platform-lifecycle-principles.md) · [17장](17-customer-center-and-safe-guide.md) · [26장](30-first-route-map-and-screen-inventory.md#12-26장-예정-27a28) · [29장](29-empty-error-permission-ux.md) · [30장 부록 A·D](30-first-route-map-and-screen-inventory.md)

> **최종 한 줄:** 운영 조치는 할 수 있지만, 그 조치가 사용자에게 **심사·보증·인증**처럼 보이게 만들지 않는다.  
> 우동공과는 **인증기관이 아니라**, 신뢰 판단에 도움이 되는 메커니즘을 제공하는 **연결 플랫폼**이다.

## 0. 대원칙 (가장 먼저 잠금)

### 0-1. 우동공과에 없는 구조

- 운영자 **심사 / 승인 / 반려** 구조 **없음**
- 우동공과는 **인증기관 아님**

### 0-2. 우동공과가 하는 것

| ○ 역할 |
|--------|
| 공부방·과외쌤·학생/학부모가 정보를 **등록·공개**할 수 있게 함 |
| 신뢰 판단에 도움이 되는 **제출자료 항목·공개 메커니즘** 마련 |
| 신고·안전 이슈·허위/도용 의심 시 **최소 운영 조치** |

### 0-3. 우동공과가 하지 않는 것

| ✕ 금지 |
|--------|
| 운영자가 내용을 심사해 **승인/반려** |
| 제출자료 **진위 공식 인증** |
| 「검증 완료」「인증됨」「공식 확인」 등 **보증 상태** 부여 |
| 소비자를 대신한 **최종 진위 판단** |

**연결 플랫폼:** 쌍방 연결 구조일 뿐. 중요 서류의 최종 진위 확인은 **소비자가 각 기관 기준으로 직접** 재확인.

## 1. 운영자 허용 조치 (최소 한정)

신고 대응·서비스 안전 유지에 필요한 **최소 운영 조치**만.

| # | 허용 조치 |
|---|-----------|
| 1 | 프로필/게시 정보 **숨김** |
| 2 | 쪽지/접촉 **일시 제한·차단** |
| 3 | 검색/노출 상태 **수동 보정** |
| 4 | 잘못 부여된 권한 **운영 보정** |
| 5 | 신고 접수 후 **임시 보호** |
| 6 | 공지/가이드 **게시·수정** |
| 7 | 운영 기록 **열람·로그** |

→ 모두 **내부 운영 조치**. 사용자에게 **심사·승인 결과**처럼 보이게 ✕.

---

## 2. 운영자 금지 조치

| ✕ 금지 |
|--------|
| 프로필·제출자료 **승인 / 반려 / 심사 통과** |
| **검증 완료 / 인증쌤 / 신뢰도 점수** 등 공식 보증·등급 |
| 제출자료 근거 **공식 인증 상태** 지정 |
| 운영자 판단을 사용자-facing에 **심사 결과**처럼 노출 |
| 제출자료에 대해 운영자가 **진위 최종 확인자**처럼 행동하는 구조 |

## 3. 제출자료 내부 확인 (A28-06)

### 3-1. 허용 목적

- 신고 대응
- 도용/불법/명백한 허위 의심 대응
- 분쟁성 이슈 확인
- 서비스 안전상 필요한 **최소 사실 확인**

### 3-2. 제출자료는 심사 대상이 아님

공개 가능 여부를 운영자가 **심사·인증**하는 구조로 사용 **하지 않음**.

### 3-3. 제출자료는 신뢰 참고자료

사업자등록증·졸업증명서·성적증명서 등 — 등록자가 공개한 **참고 정보**.

| ✕ 아님 |
|--------|
| 운영자 검증 후 붙이는 공식 상태 |
| 플랫폼 인증 구조 |
| 소비자 대신 최종 확인 |

### 3-4. 최종 확인 책임

중요 서류는 필요 시 **소비자가 발급기관 기준으로 직접** 재확인.

### 3-5. 내부 확인 → 사용자 화면 번역 금지

- 검증 완료 · 인증됨 · 플랫폼 확인 완료 · **운영자 확인 완료** · **공식 인증**

### 3-b. 노출·권한 수동 보정 (A28-07) — **정책·액션·로그 잠금** (2026-07-07)

**역할:** [§1 #3 검색/노출 상태 수동 보정](28-admin-console-red-line.md#1-운영자-허용-조치-최소-한정)의 **1차 구현 SSOT**  
**화면:** `#/admin/exposure` · **API:** `GET/PATCH /api/admin/exposure.php`  
**코드 정본:** `AdminExposureService` · `admin-red-line-copy.js` `A28_07_*` · `a28-copy.js` UI 라벨

#### 3-b-1. 범위 (1차)

| 대상 `target_type` | 노출 축 | 2차 축 (공부방만) | A28-07 1차 |
|--------------------|---------|-------------------|:----------:|
| `study_room` | `profile_status` | `inquiry_status` | ✅ |
| `tutor` | `profile_status` | — | ✅ |
| `submission` (`board_key=submission`) | `status` | — | ✅ |
| `student` | `exposure_status` | — | 후순위 |

**운영형 상태 라벨만 사용** (심사형 용어 ✕): 숨김 · 공개중 · 비공개 · 게시중 · 제출됨 · 저장중

| DB 값 (`profile_status` / `status`) | 운영 UI 라벨 | 사용자-facing 금지 |
|-------------------------------------|--------------|-------------------|
| `published` | 공개중 / 게시중 | 승인됨 · 검증 완료 |
| `hidden` | 숨김 / 비공개 | 반려 · 거절 |
| `draft` | 비공개(저장) | 심사 대기 |
| `submitted` | 제출됨 (submission만) | 검수 대기 |

#### 3-b-2. API `action` 잠금표 (PATCH)

| `action` | `target_type` | DB 반영 | `action_kind` (로그) | `user_notified` 기본 |
|----------|---------------|---------|----------------------|:--------------------:|
| `hide` | `study_room` · `tutor` | `profile_status` → `hidden` | `hide_profile` | **true** |
| `hide` | `submission` | `status` → `hidden` | `submission_hide` | **true** |
| `publish` | `study_room` · `tutor` | `profile_status` → `published` | `exposure_correction` | **false** |
| `publish` | `submission` | `status` → `published` (`submitted` **제외** → A28-06) | `submission_expose` | **false** |
| `inquiry_status` | `study_room` **만** | `inquiry_status` → 값 지정 | `exposure_correction` | **false** |

`inquiry_status` 허용 값: `open` · `paused` · `capacity_full` · `waiting_only` (20장 2축 · 검색 노출과 별개)

**금지 `action` (본 단계):** `approve` · `reject` · `verify` · `audit_pass` — **구조·UI·API 모두 ✕**

#### 3-b-3. 운영 로그 매핑 잠금표 (§9 연동)

| `action_kind` | `target_type` | `target_id` 형식 | `reversible` | 비고 |
|---------------|---------------|------------------|:------------:|------|
| `hide_profile` | `study_room` · `tutor` | 숫자 ID | true | §10 알림 가능 |
| `exposure_correction` | `study_room` · `tutor` | 숫자 ID | true | publish · inquiry 보정 |
| `submission_hide` | `board_post` | `submission:{post_key}` | true | 제출 숨김 |
| `submission_expose` | `board_post` | `submission:{post_key}` | true | 제출 게시중 |

**공통 PATCH 필드:** `reason_category` (선택) · `internal_memo` (선택 → `detail_memo` · submission은 `board_posts.internal_memo` 동시 갱신)  
**운영자:** 세션 `email` — 클라이언트 `operator_id` **신뢰 ✕**

#### 3-b-4. A28-06과 경계 (혼용 금지)

| 구분 | A28-06 `#/admin/submission-docs` | A28-07 `#/admin/exposure` |
|------|----------------------------------|---------------------------|
| 목적 | `submitted` **큐** 내부 확인 후 1차 반영 | 등록된 항목 **노출 상태 수동 보정** |
| 대상 상태 | `submitted` 중심 | `published` · `hidden` · `draft` · `submitted` 필터 |
| API | `/api/admin/submission-queue.php` | `/api/admin/exposure.php` |
| 조치명 | 노출 반영 · 숨김 | 숨김 · 공개중(보정) · (공부방) 상담 보정 |

**`submitted` → `published`:** **A28-06 큐 `expose`만** · A28-07 `publish` on `submission` **422 거부**  
`published`/`hidden` 보정은 A28-07 · 로그 `action_kind`로 구분. 사용자-facing에는 **심사 결과처럼 보이지 않게** (§10).

#### 3-b-5. 목록 조회 (GET) 잠금

| 쿼리 | 값 | 비고 |
|------|-----|------|
| `target_type` | `all` · `study_room` · `tutor` · `submission` | 기본 `all` |
| `status` | `published` · `hidden` · `draft` · `submitted` · 빈값=전체 | 유형별 해당 컬럼 필터 |

**접근:** `role_type=admin` 세션 필수 · `#/admin/exposure` UI 가드

## 4. 제출자료/신뢰정보 사용자-facing 표현

운영자 **판단 결과**가 아니라 **등록·공개 사실**만 표시.

| ○ 가능 | ✕ 금지 |
|--------|--------|
| 사업자등록증 공개 · 졸업증명서 공개 · 성적증명서 공개 | 검증 완료 · 인증됨 · 공식 인증 |
| 제출자료 N개 공개 · 신뢰정보 공개 | 운영자 확인 완료 · 플랫폼 보증 |
| 등록된 항목 N개 · 공개된 제출자료 N개 | 신뢰도 점수 · 인증쌤 |

### 안내 문구 (취지)

> 제출자료는 등록자가 공개한 참고 정보입니다.  
> 우동공과는 해당 서류를 인증하거나 보증하지 않으며, 중요한 서류는 필요한 경우 발급기관 기준으로 직접 다시 확인해 주세요.

**코드:** `SUBMISSION_DOC_USER_NOTICE` · `lifecycle-copy.js` `TRUST_PLATFORM_DISCLAIMER`

## 5. 문의 vs 신고

### 5-1. 문의

서비스 이용 · 정책 · 버그 · 운영 문의 → **답변/안내** 중심

### 5-2. 신고

허위 · 도용 · 부적절 접촉 · 안전 이슈 · 약관 위반 · 사기/위험 정황  
→ 숨김 · 접촉 제한 · 노출 제한 · 임시 보호

### 5-3. 초기 화면

같은 운영 화면에서 **시작 가능** · **처리 큐·조치 기준 분리** 필수.

## 10. 사용자 알림

- **항상 발송 ✕**
- 이용 상태에 **직접 영향** 있는 조치만 제한적 발송

| 알림 가능 | 알림 불필요 가능 |
|-----------|----------------|
| 숨김 · 쪽지/접촉 제한 · 이용 상태 직접 영향 · **제출 숨김** (`submission_hide`) | 내부 노출 보정 · 정합성 수정 · **제출 게시중 복원** (`submission_expose`) · 공부방 `inquiry_status` 보정 |

**알림 문구 금지:** 승인되었습니다 · 반려되었습니다 · 검토 완료 · 인증되었습니다  
**가능:** 숨김 처리되었습니다 · 일부 기능 이용 제한 · 공개 전 확인 필요
