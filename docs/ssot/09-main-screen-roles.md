## 1. 목적

메인화면은 단순 첫 화면이 아니라 **지역 단위 홈 + 공급자 노출 상품 + 역할별 탐색 시작점**을 동시에 담는다.

1차에서 먼저 잠그는 항목:

- 비로그인 메인 구조
- 로그인 후 **역할별 첫 화면**
- 공부방 / 과외쌤 / 학생(학부모)의 메인 노출 방식
- 광고 박스 / 리스트 구조
- 독점권과 슬롯 수 운영 원칙

## 4. 비로그인 메인 (§3 잠금)

### 4-1. 기본 성격

빈 화면이 아니라 **기준 지역 1곳**을 정해 **샘플 데이터**를 보여주는 **데모 홈**.

예: 강남구 대치동 · 해운대 우동

### 4-2. 구성

- 기준 지역 **지도** 및 지역 요약 수치
- 해당 지역 **대표 공부방** 노출
- 해당 지역 **대표 과외쌤** 노출
- 해당 지역 **학생 의뢰** 일부 노출
- **가입 유도 CTA**

→ 목적: **서비스 구조를 직관적으로 보여주기**

### 4-3. 진입

```
/  →  비로그인 메인 (샘플 지역 데모)
```

헤더: 로그인 · 회원가입 ([2장 §3](02-registration-and-member-db.md#3-인증가입-화면-잠금) · `preview/auth-ui/`)

## 5. 로그인 후 역할별 첫 화면 (§4 잠금)

### 5-1. 학부모 (`guardian_student`)

상단 메뉴 근처 **두 축 전환**:

| 탭 | 기본 |
|----|------|
| **우리동네 공부방** | ✅ **디폴트** |
| **우리동네 과외쌤** | 전환 가능 |

- 단일 고정 화면 **아님** → **공부방/과외쌤 탐색 전환형 홈**
- 공부방 탭: 지도 제공 · 과외 탭: 지도 **미제공**

### 5-2. 공부방 (`study_room_owner`)

메인 **두 축**:

- **공부방 박스**
- **공부방 리스트**

| 원칙 | 내용 |
|------|------|
| 학생 리스트 | 메인 **핵심 아님** (운영자 ≠ 학생 등록 주체) |
| 목적 | **내 지역 공부방 노출 구조 확인** · **지역 홈 탐색** |
| 지도 | 제공 |

### 5-3. 과외쌤 (`tutor`)

메인 **세 축**:

- **과외쌤 박스**
- **과외쌤 리스트**
- **학생 리스트**

| 원칙 | 내용 |
|------|------|
| 학생 리스트 | 메인에서 **충분히 드러남** (핵심 목적 = 학생 탐색) |
| 지도 | **미제공** |
| 성격 | **과외쌤 노출 + 학생 탐색 홈** |

### 5-4. 로그인 진입 규칙

```
로그인 성공 → 마지막 사용 역할 또는 user_roles 기준 기본 홈
```

| 우선 조건 | 기본 홈 |
|-----------|---------|
| `study_room_owner` 단독 또는 최근 사용 | 공부방 메인 |
| `tutor` 단독 또는 최근 사용 | 과외쌤 메인 |
| `guardian_student` | 학부모 메인 (디폴트: 공부방 탭) |
| 복수 역할 | **1역할 화면** + 전환 UI |

### 5-5. 가입 완료 직후 (2장 연동)

```
회원가입 완료
  → [ {회원구분} 상세등록 ]   CTA
  → [ 메인 홈으로 ]           9장 해당 역할 메인
```

플로우: **회원 구분 선택 → 공통 기본정보 → 가입 완료 → 상세등록 유도** (§12-3)

## 6. 메인 바디 구조 (§5 잠금)

**상단 3박스 / 중단 5박스 / 하단 전체 리스트** — 3층 구조.

| 영역 | 규격 | 성격 |
|------|------|------|
| **상단** | 가로 **3** 박스 | 최강 노출 · 공부방/과외쌤 **최상위 노출 상품** |

## 8. 독점권·슬롯 수 운영 (§7 잠금)

### 8-1. 기본 판단

상단 독점/박스 = **실제 슬롯 수 고정**.  
화면만 독점처럼 보이고 같은 위치 상품 **무제한 판매** → 1차 **부적절**.

### 8-2. 이유

- 광고 의뢰 공급자는 **구매 박스가 홈에 보이는지** 즉시 확인
- 공급자 로그인 시 **본인 구매 박스 미노출** → 신뢰 하락
- **보이는 수 ≈ 실제 판매 수** → 신뢰·운영 단순성

### 8-3. 추천 운영안

#### 상단 3박스

- **실제 3칸 고정** · 지역(단지/동) 기준 슬롯 고정
- 독점권 / 상단 고정 노출 상품
- **구매자 로그인 시 해당 박스 즉시 노출**

#### 중단 5박스

- 1차: **5칸 고정**
- 추후: 순환 · 시간대별 · 요일별 **회전형** 검토

#### 하단 리스트

- 지역 **전체** · **최근 등록** 우선
- 고정 박스형보다 **기본 노출 영역** 성격

### 8-4. 1차 결론

| 영역 | 운영 |
|------|------|
| 상단 | **실제 고정 슬롯** |
| 중단 | **우선 고정** → 추후 회전형 |
| 하단 | **전체 리스트** |

> 노출 상품 DB는 공부방 **기본 정보**(5장)와 **분리** (5장 §10)

## 9. 반응형 웹 (§8 잠금)

**대전제: 반응형 웹** — 동일 상품 구조 · **배치만** 조정.

### 9-1. 데스크톱

- 지도 + 상/중/하단 구조
- **우측 광고 슬롯** 운영 가능

### 9-2. 모바일

- 우측 광고 슬롯 **제거**
- 본문 **인라인** 광고/프로모션 카드
- 상단 3 / 중단 5 → **스와이프** 또는 **세로 카드 묶음**

## 10. 인트로·로그인·회원가입 (§9 잠금)

### 10-1. 인트로 화면

- 기존 인트로 시안 = **서비스 소개형 참고안**
- 학생/공부방/과외쌤 3축 아이디어 **유지 가능**
- **1차 메인 홈 기준안으로는 사용하지 않음** (지역 탐색·역할별 진입 우선)

### 10-2. 로그인

- **9장 부록** 레이아웃: 흐린 홈 실루엣 · 중앙 카드 · 유틸 바(지역·이용안내·고객센터) · **카드 아래** 목적 칩 3개
- 빠른 로그인·가입·소셜 진입 우선 · 성공 시 **home-ui** 복귀 (`return_to` 지원)
- 소셜 **3축 (한국)**: **네이버 → 카카오 → 구글** — `user_oauth_accounts` · `/api/auth/oauth/`
- **인증정책·로그인 부록**: [9장 부록](09-appendix-login-and-auth-policy.md) §16 — 이메일+비밀번호 1차 · 행동 전 인증 · 비밀번호 규칙 · 메일 링크 재설정
- **소셜 신규 가입**: OAuth 콜백 → `#/signup/role?from=oauth` → 역할 선택 → (공급자) 기본등록
- SSOT UI: `preview/auth-ui/` · 상세: **[9장 부록](09-appendix-login-and-auth-policy.md)**

### 10-3. 회원가입

- **회원 구분 선택** → **공통 기본정보** → **가입 완료** → **상세등록 CTA**
- 역할 카드: 학생(학부모) / 공부방 / 과외쌤 (2장 §3)

## 11. 광고 배너 (§10 잠금)
