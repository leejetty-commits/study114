## 1. 목적

17장은 **고도화된 상담·티켓 시스템을 1차 목표로 하지 않는다.**  
1차는 **설명형 허브**와 **정적 안내** 중심이며, 법률 자문·결제 중개·에스크로 기능을 제공하는 장이 **아니다.**

| 축 | 내용 |
|----|------|
| **경계 선언** | 에스크로 · 지급 보류 · **안전번호(플랫폼 중계)** · 전화/메일 **플랫폼 노출** 하지 않음 |
| **안전과외** | 선입금 주의 · 분쟁 예방 · 당사자 책임 — **교육형 가이드**(6장 §2) |
| **접촉 안내** | **회원 간** 공식 접촉 = **쪽지(16장)** · **운영 문의** = 고객센터 채널(§7) |
| **이용안내** | 서비스 구조 · 역할 · 메뉴 · 1차 범위 요약 |

17장은 **새 정책을 만드는 장이 아니라**, 4·6·13·16장 등 **이미 잠근 정책을 사용자-facing 문구·화면으로 정리**하는 장이다.

### 1-1. 6장과의 관계 (잠금)

|----------|-----------|
| 안전과외 **GNB 독립 메뉴 제외** | GNB에는 **「고객센터」만** · 안전과외는 **하위 콘텐츠** |

## 3. 안전과외 원칙 (정본 카피)

P17-01 히어로 · P17-03 상단에 **긍정 3원칙 + 부정 5항**을 함께 노출한다.

### 3-1. 3대 원칙 (긍정)

| 원칙 | 사용자-facing 방향 |
|------|-------------------|
| **학생 보호** | 민감정보·공개 범위를 보수적으로 — 필요한 범위만 공개 |
| **공급자 검증** | 등록·증빙·검토 상태를 확인하며 비교 |
| **플랫폼 비중계** | 대금·연락 **중개·보증**은 1차 핵심 기능 아님 |

### 3-2. 「우동공과가 하지 않는 것」(부정)

| 항목 | 사용자 메시지 |
|------|---------------|
| 결제 **에스크로** · 지급 보류 | 「대금은 당사자가 직접 합의·결제」 |
| **안전번호**(플랫폼 대리 연락) | 「플랫폼이 번호를 중계하지 않음」 |
| 전화·이메일 **플랫폼 노출** | 「회원 간 **공식** 접촉 = **쪽지**」 |
| **매칭 알고리즘** · 퀵매칭 | 「탐색·비교는 회원 주도」 |
| 법률·분쟁 **대리 해결** | 「당사자 협의 · 필요 시 관할 기관」 |

> **안전번호:** 4·14장 — 필드 **숨김·미수집**. 17장은 **비제공**만 안내.

### 3-3. 「공식 접촉 = 쪽지」 적용 범위

| 구분 | 채널 | 장 |
|------|------|-----|
| **회원 ↔ 회원** 공식 접촉 | **쪽지** | 16장 |
| **운영·서비스 문의** | 이메일 / 외부 폼 (P17-07) | 17장 |
| 전화·플랫폼 메일 노출 | ✕ | 6장 |
