# 프로젝트 개요 — study114 (우동공과)

## 목적

동네 단위로 **공부방**과 **과외쌤** 정보를 등록·검색할 수 있는 플랫폼.
1차 홈 핵심은 **공부방**, 과외쌤은 등록·기본 검색까지.

## 기술 기준

| 항목 | 기준 |
|------|------|
| 언어 | PHP 8.2 |
| DB | MySQL 8 |
| 문자셋 | UTF-8 (`utf8mb4` / `utf8mb4_unicode_ci`) |
| 배포 | 로컬 개발 → study114.net |

## 1차 포함

- 가입 방식 설계
- 공통 회원 DB
- 공부방 등록 DB
- 과외쌤 등록 DB
- 홈/서브 페이지로 확장 가능한 구조 준비

## 1차 제외

- 상품·결제 설계
- 만 14세 미만 가입 분기
- 과외쌤 홈 핵심 노출 (검색·등록만)

## 핵심 설계 원칙

### 1. 회원 축 — 학부모 기준

- 학생·학부모를 **하나의 회원 축**으로 본다.
- 계정 주체는 **학부모(보호자)** 기준.
- 자녀(학생) 정보는 회원에 **종속**되는 별도 엔티티로 관리.
- 1차에서는 만 14세 미만 특수 가입 절차 없음.

### 2. 한 회원, 복수 역할

- 동일 `users` 계정이 **공부방 운영자**와 **과외쌤** 역할을 **동시에** 가질 수 있다.
- 역할은 `user_roles`로 관리 (M:N 아님, 한 user에 여러 role row).

### 3. 공부방 데이터

- **가격**: 숫자형(`price_amount`) + 설명형(`price_description`) 병행.
- **시설/환경**: 체크형(마스터 + 선택) + 자유기술형(`facility_note`).

### 4. 지역 기준

- 아파트/단지 정보가 있으면 **단지 우선**.
- 단지가 없으면 **동(dong) 기준**.
- 상세: [04-location-policy.md](04-location-policy.md)

### 5. 확장성

- URL·라우팅은 `/`(홈), `/study-rooms`, `/tutors` 등 **서브 독립** 구조.
- 공통 회원·지역 테이블을 공부방·과외가 **공유**.

## 용어

| 용어 | 설명 |
|------|------|
| 회원(user) | 로그인 계정. 1차 기본 주체는 학부모 |
| 학생(student) | 회원 소속 자녀 정보 (검색·매칭용, 1차는 최소 필드) |
| 공부방(study_room) | 회원이 등록하는 학원·공부방 정보 |
| 과외쌤(tutor) | 회원이 등록하는 과외 선생님 프로필 |
| 단지(complex) | 아파트·주상복합 등 물리 단지 |
| 동(dong) | 행정동 또는 법정동 (정책에 따라 코드 테이블로 통일) |
